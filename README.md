# ERD Exercise Questions

Three ER Diagram design exercises across different industries. Each question presents a real-world business scenario. Students must identify entities, attributes, relationships, and cardinalities, then draw a complete ER diagram.

| # | File | Industry | System |
|---|------|----------|--------|
| 1 | [Q1_Hospital_Management.md](Q1_Hospital_Management.md) | Healthcare | Hospital Management System |
| 2 | [Q2_Ecommerce_Platform.md](Q2_Ecommerce_Platform.md) | Retail | Online E-Commerce Platform |
| 3 | [Q3_Hotel_Booking.md](Q3_Hotel_Booking.md) | Hospitality | Hotel Booking System |

---

## What to Submit

- ER Diagram drawn using any tool (draw.io, Lucidchart, pen & paper scanned)
- Use **Chen notation** or **Crow's Foot notation** (confirm with your instructor)
- Label all entities, attributes, primary keys, relationships, and cardinalities

---

## Marking Criteria (Per Question)

| Criteria | Marks |
|----------|-------|
| Correct entities identified | 10 |
| Correct attributes with primary keys | 10 |
| Correct relationships & cardinality | 15 |
| Weak entities / relationship attributes | 10 |
| Diagram clarity and notation | 5 |
| **Total per question** | **50** |
